- No empty routes.

## Architecture

Each module should already have:

- Dashboard

- Tables
- Empty states

Build the UI architecture first.

## Data

Whenever Neon already contains real data that is safe to reuse, use it.

If data does not yet exist, create realistic placeholders that follow the same structure as the Reporting module.

Do NOT duplicate Reporting logic.

Instead, build reusable shared components wherever possible.
