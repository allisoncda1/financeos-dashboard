# Database Rules

Every QBO object must first be written into:

Then normalized into:

- accounts
- customers
- vendors
- invoices
- bills
- transactions

No duplicate records.

UPSERT only.

Soft delete only.

Never delete accounting history.

# Validation

After each entity completes:

Run the FinanceOS Validation Engine.

Store results in:

validation_results

Record:

- total rules
- passed
- failed
- execution time

If validation fails:

DO NOT stop the backfill.

Mark the run as partial.

Continue with remaining entities.

Once the historical backfill succeeds:

Implement incremental synchronization.

Use:

LastUpdatedTime

for every supported object.

The sync engine should only request modified records.

Never re-download the full history unnecessarily.
